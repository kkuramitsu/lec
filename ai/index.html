    データサイエンス・AI入門#

    公開前

    本資料は、正式な公開前で内容やコードのチェックがまだ完全に終わっていません。

    データサイエンス・AIは、これからの大学生や高校生が学部や専門を問わず、 学ぶべき内容として注目を集めてみる内容です。

    本講義ノートは、日本女子大学の講義・演習向けにまとめたノートです。

    • 数物情報科学科（３年生前期）
    • AI入門（全学部全学科）

    最近は、データサイエンスや人工知能を学んでみたいという学生、 もしくは、（これも不思議な話ですが）学ばせて欲しいという先生が増えて、 基礎から積み上げてみたいな前提が成り立たなくなっています。

    本講義ノートは、数学が高校レベルの理解であっても、仮にPythonがほぼ未習者であっても、 データサイエンスやAIを体験的に入門できるように配慮してまとめてみました。 （若干というかかなり無理が残っていますが..） データサイエンスやAIは面白い！と感じてもう少し学びたいと思ったら、 是非、基礎に戻って、Pythonやエンジニアリング力をあげてください。

    一応、自習で使う読者のために、難易度を示しておきます。

    • ⭐️: 少々、アドバンスな内容（飛ばしても構わない）
    • ⭐️⭐️: Python未習者は飛ばすべき内容

    講義ノートは、自分の備忘録を含めて代わりに色々書いてあります。 実際の講義や演習では、履修者の理解と進捗にあわせて、 補足説明したり、スキップしたりしながら進めます。

    そもそも入門する前に#

    データサイエンスやAIを学ぶということはどういうことでしょうか？

    コンテストの審査委員

    日の出町では、愛犬コンテストが開催されることになりました。 複数の審査員がそれぞれ10点満点で採点して、その平均点を計算することで、 優勝のワンちゃんを決めることになりました。

    皆さんは、小学校から算数を学んできました。 だから、突然、コンテストの集計係を任されても問題なく、 うろたえることなく業務を遂行できるはずです。

    例えば、５人の審査員が、８点、８点、９点、７点、５点と採点したとき、 平均値は次のように求めることができます。

    もし集計すべきデータが増えたらどうなるでしょう。 作業量も増えて膨大な時間がかかります。 また、人間による手作業は、仮に電卓を使ったとしても、間違いが多くなります。

    日本はデジタル化が遅れている という話をよく聞くかと思います。 古き良き日本社会は、こんな感じでした。

    • 計算間違いしないように頑張りましょう。
    • ダブルチェック、大切！
    • 手作業で処理できないことは 諦めましょう。

    簡単に諦めちゃっていたから、デジタル化の進んだ国に差を広げられた訳ですね。

    それでは、データサイエンスやAIを学ぶと、どう変わるべきなんでしょう？

    • 計算は、もうコンピュータで処理しましょう
    • 間違えたくなければ、コンピュータで処理しましょう
    • もうコンピュータで処理して、新しい可能性に挑戦しましょう

    最後が、一番大切です。 諦めるのではなく、新しい可能性に挑戦することです。

    データサイエンスやAIは、コンピュータを使いこなすことが前提の学問です。 コンピュータを使いこなせてこそ、習った知識も有効に活きてきます。 だから、コンピュータを自由自在に使いこなして 新しいことに挑戦して欲しいです。

    プログラミングについて#

    プログラミングは、コンピュータを自由自在に使いこなす基本スキルです。 本講義では、プログラミング言語としてPython言語を使います。

    データサイエンスやAIに取り組むためには、 どの程度、Python をマスターしておくべきか、少し心配かもしれません。

    まず、先ほどの審査委員に例に戻って平均値を求めてみましょう。

    ５人の審査員が、８点、８点、９点、７点、５点と採点したとき、

    Pythonで平均値を求める
    mean = (8 + 8 + 9 + 7 + 5) / 5
print(mean)

    7.4

    Pythonをはじめプログラミング言語は、数式や簡単な英語でコードを書けるように 設計されています。少しコツをつかめば、プログラミングはそれほど高い壁にはなりません。

    また、現在のプログラミングは、便利なライブラリが整備されています。 データサイエンスやAIの分野では、このようなライブラリを正しく使うだけで、 数式を理解してプログラミングしなくても、目的の処理ができます。

    平均値を求める
    import numpy as np
a = np.array([8, 8, 9, 7, 5])
print(a.mean())

    7.4

    英語は大切

    Python は海外で開発されているため、ライブラリは英語になっています。 結構、英語の語彙力はプログラミングを理解するときのヒントになります。 （ちなみに、平均はavarageではなく、meanです。）

    数式について#

    数式は、どうも苦手だという人が多いと思います。 しかし、データサイエンス・AI入門では、数式を使った説明がよく出てきます。

    標準偏差は、例えば、次のように数式で定義して与えられます。

    • (ミュー)は、ギリシア文字のひとつで、英字にに相当します。平均は、mean なのでその頭文字からきています。
    • (シグマ)は、ギリシア文字のひとつで、英字のに相当します。総和は、summantionなので、その頭文字からを使います。

    まず、「標準偏差」の意味を理解しようと思って、数式を読んではいけません。 標準偏差とは、平均値を中心にほとんどのデータが含まれる上下の範囲を示していますが、 それを数式から読み取るのは無理です。 海外の数学者が、の式で計算される値を standard deviation と呼んで、 日本人の数学者が標準偏差と訳しただけです。

    数式は、ほとんどの場合、計算方法を示す定義として用いられます。

    Pythonはライブラリが豊富に揃っているため、 数式からコードを書かなければならない機会はほとんどありません。 Python なら、数列a の標準偏差は、a.std()と書けば計算できます。

    標準偏差を求める
    import numpy as np
a = np.array([8, 8, 9, 7, 5])
print(a.std())

    1.3564659966250536

    メソッド名 std()の由来

    これがわかれば、プログラミングは得意になれるでしょう。

    このような理解は、数式を見ただけで理解できるものではありません。

    学習と機械学習#

    ニュートンは、リンゴが落ちるのを見て、万有引力の法則を発見しました。 もう少し正確に書くと、いろんな現象の観察結果と微積分を使って、「 としか思えない」という理由を示しています。

    発見された万有引力の法則は、数理モデルとして記述されて、これから発生する事象（未来）を予想できるようになります。

    数理モデル

    ニュートンが万有引力の法則を発見に至ったプロセスは興味深いものです。このプロセスをアルゴリズム化して、コンピュータが法則を発見し、モデル化する手法が機械学習です。

    種類:

    • 教師なし学習 (K-means法, PCAなど)
    • 教師あり学習 (線形回帰, ロジスティック回帰、決定木、サポートベクターマシン)
    • 強化学習 (Q-Learning)

    実は、前回までに学んだK-means法は教師なし学習、線形回帰は教師あり学習と分類され、機械学習アルゴリズムの一種でした。人工知能の基礎となる機械学習といっても、身構えなくてはならないほど難しいものではありません。

    教師あり学習#

    教師あり学習は、問題と答えをコンピュータに与えることで予測モデルを学習させる方法です。 事前に、「特徴を表すデータ」と答えである「目的データ」が必要です。

    説明のため、これまでに作成してきた身長体重データセットを思い出してください。

    • 説明変数: 特徴を表すデータとして、「身長」と「体重」が与えられる
    • 目的変数: 目的データとして、「職業」が与えられる

    学習

    機械学習とは、説明変数と目的変数の大量のデータセットを与え、そこから目的変数を予測するモデルを作成することです。

    予測 

    予測モデルに未知のデータを与えると、目的データが出力されます。

    目的データの種類によって、回帰問題と分類問題に分けられる。

    Note

    回帰と分類

    • 回帰問題: 数値を予測すること
    • クラス分類問題: 職業などのカテゴリを予測すること

    クラス分類#

    クラス分類は、カテゴリデータを予測することです。 カテゴリの数が２つなら２クラス分類、それ以上なら多クラス分類と呼ばれます。

    クラス分類が重要なのは、画像認識など興味深いアプリケーションが作れる点にあります。 例えば、画像を多次元データとして説明変数、そのラベルを目的変数として学習させると、 画像認識が可能になります。後半では、簡単な画像認識を少しだけ学びます。
